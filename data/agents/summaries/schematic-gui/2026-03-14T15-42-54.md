# Collapsible category tree added to Components palette

## What changed
- **Empty search state** now shows a full component tree instead of "Type to search components"
- Components are grouped into: **Power**, **Passives**, **Diodes**, **Transistors**, **Op-Amps**, **ICs**
- Each category row has an icon, label, and a count badge showing how many components are in it
- Click a category header to **collapse / expand** it — state is saved in `localStorage` so it remembers your preferences across page reloads
- Items inside each category are sorted alphabetically
- **Typing in the search bar** still works exactly as before — shows a flat filtered list
- New `_SCH_CATS` lookup maps every `symbol_type` to the right category automatically (so new components added to the library slot into the correct group without code changes)

## Files
- `frontend/static/js/index-navigation.js`

## Notes
- Grouping uses the `symbol_type` field from each component's profile. Any component with an unknown type falls back to the "ICs" group.
- The ⚡ "Load example" button still appears on every item in both tree and search modes.

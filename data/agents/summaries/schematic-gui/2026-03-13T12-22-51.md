# Fix: components connected to wire interior no longer drag the wire

## What changed
- Added `_isDotAtInterior(wire)` helper that returns `true` when a wire is a zero-length dot-wire sitting on the *interior* of another wire segment (a T-junction marker created by `_autoConnectPorts`)
- `_down()` now skips these T-junction dot-wires when building `connectedWires` for single-component drags
- `_buildGroupDragItems()` applies the same skip for group/multi-selection drags

## Files
- `frontend/static/js/index-schematic-editor.js`

## Notes
The root cause: `_autoConnectPorts` stamps a dot-wire `{points:[{x,y},{x,y}]}` at every T-junction. Both endpoints of that dot-wire matched the port within TOL, so both got added to `connectedWires` and moved with the component. The actual wire spanning the junction never moved, but the junction marker did — breaking the visual and the net connection. Now those T-junction markers are identified and excluded from dragging.

# Fixed NET:— wires in PMA3-83LNW+ Application Circuit

## What changed
- **Backend amplifier ports fixed**: `_SYMDEFS["amplifier"]` had wrong offsets (`in` at −50,−20 and `in2` at −50,+20) that didn't match the frontend single IN port at (−50, 0). Changed to `IN(−50,0)`, `OUT(+50,0)`, `GND(0,+40)`.
- **Saved project regenerated**: `mmi68vrwylous.json` was a stale version with wrong component positions and missing 3 components (C1, VDD2, GND3) and 5 wires. Rebuilt from the canonical `buildExampleCircuit()` definition (11 components, 13 wires, 2 labels).
- **Net label positions corrected**: RF_IN and RF_OUT labels were at `y=220` but the wire endpoints are at `y=240`, so they never attached. Fixed in both the saved project and `buildExampleCircuit()` in `index-navigation.js`.

## Result
All 13 wires now show proper net names: `RF_IN`, `RF_OUT`, `GND`, `VCC`, `GND_P2` (signal input node), `VCC_P1` (bias/output node). No wire shows `NET:—` any longer.

## Files
- `backend/schematic_api.py` — fixed `_SYMDEFS["amplifier"]`
- `frontend/static/projects/mmi68vrwylous.json` — regenerated circuit data
- `frontend/static/js/index-navigation.js` — fixed label positions in `buildExampleCircuit()`

## Notes
The `GND_P2` name on the RF signal input net (U1.IN, C2.P2, L1.P2) is auto-generated by passive propagation (L1 bridges the signal node to GND). Adding a net label on the signal junction wire would give it a friendlier name like `RF_SIG`.

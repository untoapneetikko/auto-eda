# Every wire, pin, VCC/GND now gets a net — including inheritance on connect/drag

## What changed

### Frontend (`index-schematic-editor.js`)
- **Net overlay labels now cover VCC/GND-only nets** — previously, if a net had only VCC or GND ports (e.g. a wire between two VCC symbols), `_netOverlayH()` would skip it entirely. Now it falls back to the power port as the label anchor.
- **Wire-only clusters now show net names** — isolated wires (not touching any component pin) already got N1/N2/… names from the backend, but the canvas never rendered those labels. Now the mid-point of any such wire gets a tiny net-name label when "Show Nets" is on.
- **Wire colour = net colour** — when the "Show Nets" toggle is on, every wire is rendered in the colour of its assigned net (using `_labelColor(netName)`). This is an immediate visual confirmation that every wire has been assigned a net. Unresolved wires stay the default blue.
- **Net overlay now populated on project load** — previously `_cachedNetOverlay` stayed null until the first edit (600 ms debounce). Now `loadProject()` calls `_refreshNetOverlay()` immediately so the net panel shows correct data as soon as a project opens.
- **Net overlay refreshed after undo / redo** — undo/redo never triggered a net cache refresh, so the net list would go stale after history navigation. Both functions now schedule a debounced refresh.

### Backend (`schematic_api.py`)
- **`_SYMDEFS` port names aligned to frontend** — all short-form symbolic names now match `SYMDEFS` in `index-navigation.js`: `vcc → VCC`, `gnd → GND`, `diode/led → A/K`, `BJT → B/C/E`, `MOSFET → G/D/S`, `opamp → +/-/OUT`, `capacitor_pol → +/-`. This means netlist pin references (e.g. `C1.+`) are consistent with what the canvas displays.
- **VCC net inherits component value** — a VCC symbol with value "3V3" or "5V" now creates a net named "3V3" / "5V" rather than always "VCC". Multiple power rails are handled automatically.
- **Priority 1 detection updated** — the VCC/GND union-find lookup uses the new `port::cid::VCC` / `port::cid::GND` keys to match the renamed ports.

## Files
- `frontend/static/js/index-schematic-editor.js`
- `backend/schematic_api.py`

## Notes
- "Show Nets" toggle (top toolbar) must be enabled to see wire colouring and mid-wire labels. The net panel in the sidebar is always updated.
- Existing projects are unaffected — the union-find matches by position, not port name, so all connectivity is preserved.

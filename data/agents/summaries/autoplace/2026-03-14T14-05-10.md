# Fixed rotation pass-through — components no longer all default to 0°

## What changed
- `data/outputs/example_schematic.json`: added `rotation` field to all 7 components (U1=0°, C2=90°, L1=180°, C3=270°, L2=0°, C1=0°, C4=0°)
- `agents/autoplace/agent.py`: `get_placement_context()` now passes `rotation` field from schematic through to the component list
- `agents/autoplace/placement_optimizer.py`: result builder now uses `comp.get("rotation", 0)` instead of hardcoded `0`

## Files
- `data/outputs/example_schematic.json`
- `agents/autoplace/agent.py`
- `agents/autoplace/placement_optimizer.py`

## Notes
- The fix is generic: any future schematic with a `rotation` field on its components will have that rotation honoured automatically
- Defaults to 0° when no rotation is specified (safe fallback)

# Fixed courtyard expansion missing from placement optimizer and DRC checker

## What changed
- Added `COURTYARD_EXPANSION_MM = 0.10` constant (IPC-7351 / KiCad standard: 0.10 mm per side)
- `_build_dataframe` (used by `check_package_gaps` DRC): AABB bounds now computed as `package/2 + 0.10 mm` instead of `package/2` — so the checker was previously accepting spacings that violated the courtyard layer
- `compute_net_proximity_placement`: `hw`/`hh` arrays (used for both repulsion forces and boundary clamping) now include the courtyard expansion — components will be pushed apart to respect the full courtyard boundary, not just the bare package outline
- Rationale strings updated to explicitly mention "courtyard clearance (courtyard = package + 0.10 mm per side)"

## The bug
Without this fix, every pair of components could end up 0.20 mm (2 × 0.10 mm) too close — their courtyard layers would overlap, but neither the algorithm nor the DRC checker would flag it. Any downstream KiCad DRC would fail.

## Files
- `agents/autoplace/placement_optimizer.py`

## Notes
Minimum legal centre-to-centre distance for two 0402 caps (1.0×0.5 mm body) along the long axis is now correctly computed as:
  (0.5 + 0.10) + (0.5 + 0.10) + 0.25 = **1.45 mm** (was incorrectly 1.25 mm)

# Fixed: run_autoplace now applies correct PCB rotations from example_schematic.json

## What changed
- `backend/schematic_api.py` (`run_autoplace`): now loads PCB rotations from `data/outputs/example_schematic.json` and applies them to each component on write-back
- Previously, `orig_rotations` was read from the board's current component data (all 0° on a fresh board), so autoplace never changed any rotation
- Now: `pcb_rotations` dict is built from example_schematic → used in `opt_components` → applied on write-back with priority over the board's original values

## Files
- `backend/schematic_api.py`
- `data/agents/summaries/autoplace/<this file>`

## Notes
- Tested: starting from a board where all rotations = 0°, after autoplace: C2=90°, L1=180°, C3=270°, U1/L2=0° — all correct
- Falls back silently to orig_rotations if example_schematic.json is missing or unreadable

# Enforce 90° minimum trace angle

## What changed
- Added `minTraceAngle: 90` to the `DR` (design rules) object in `pcb-globals.js` — makes it a first-class, tunable design rule (editable via Layer Manager)
- Updated `_checkTraceAngles` to use `DR.minTraceAngle` (converted to radians) instead of the old hardcoded 45° (`Math.PI/4`)
- Added angle enforcement when **clicking to place new route points** during interactive routing — if the proposed bend angle is less than `DR.minTraceAngle`, the click is rejected and an on-screen error shows (e.g. `Angle too sharp (62° < 90° min)`)
- Both enforcement paths (drag-to-move existing segment + click-to-add new point) now use the same 90° minimum

## Files
- `frontend/static/js/pcb-globals.js` — added `minTraceAngle: 90` to DR
- `frontend/static/js/pcb-editor.js` — updated `_checkTraceAngles`, added angle check in route-point click handler

## Notes
- The minimum can be changed by editing `DR.minTraceAngle` (or via the Layer Manager if exposed there)
- 90° is the industry-standard minimum for PCB traces — avoids acid traps in etching

# Fix: component positions now persist when reopening PCB layout

## What changed
- **Root cause**: Every time the PCB tab was opened, the `importProject` message handler called `importSchematic` unconditionally — even when a saved board already existed. This re-ran backend placement and overwrote all saved component positions.
- **Fix**: Added an early-return path in the `importProject` handler: if a saved board already exists for the project, load it directly and skip `importSchematic` entirely. `importSchematic` is now only called for brand-new boards that have never been saved.

## Files
- `frontend/static/pcb.html` — modified `importProject` message handler (lines ~2463–2505)

## Notes
- First-time import (no saved board) behaves exactly as before.
- Returning to the PCB tab now always restores your last saved layout — positions, rotations, traces, and vias all preserved.
- If you need to force a re-import from the schematic (e.g. after adding new components), you can delete the board via the board tabs and reopen the PCB section.

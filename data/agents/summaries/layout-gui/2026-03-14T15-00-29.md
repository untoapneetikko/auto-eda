# Fix group rotation for both PCB editor and Layout Example

## What changed
- **Root cause fixed**: `rotateSelGroup` was calling `_snapshot()` before the mutations. In the embedded (Layout Example) iframe, the `_snapshot` wrapper immediately fires `leBoardChanged` to the parent — with the *pre-rotation* board. The parent cached that old state, so the rotation appeared briefly then reset. Fix: mutations now happen first, then `_snapshot()` is called so the notification carries the post-rotation board.
- **Context menu**: added "Rotate N components 90° CW" to the right-click menu for multi-selections (it was keyboard-only before).
- **Info panel**: when multiple components are box-selected (`selectedComp=null`), now shows a dedicated multi-selection panel with a ↻ Rotate button, component count, and Del all — previously showed the unhelpful "Click a component" placeholder.

## Files
- `frontend/static/js/pcb-editor.js` — snapshot order fix + context menu rotate entry
- `frontend/static/js/pcb-panels.js` — multi-selection info panel

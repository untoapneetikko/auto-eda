# Work layer selection now drives all drawing tools in PCB + Example Layout

## What changed
- **`_setWorkLayer(name)`** new helper in `pcb-panels.js`: sets `editor.workLayer` AND syncs `layers[k].active` on every layer simultaneously — previously only `workLayer` was set, so zone/area tools (which read `layers[k].active`) always defaulted to F.Cu
- **`openLayerPickerDropdown()`** in `pcb-panels.js`: a compact floating layer-picker dropdown attached to the toolbar `Layer:` badge. Works in both `?app=1` (main PCB) and `?embedded=1` (Layout Example iframe). Replaces the broken `scrollIntoView()` call that silently failed inside embedded iframes
- **`pcb.html` work-layer-badge**: onclick now calls `openLayerPickerDropdown()` — click the current layer name in the toolbar to get an instant layer picker anywhere
- **`pcb-editor.js` zone tool**: layer assignment now reads `this.workLayer` instead of `layers[k].active`
- **`pcb-editor.js` area tool**: same fix

## Files
- `frontend/static/js/pcb-panels.js`
- `frontend/static/js/pcb-editor.js`
- `frontend/static/pcb.html`

## Notes
- Trace routing already used `workLayer` correctly — only zone/area were broken
- The dropdown closes when clicking outside or selecting a layer
- The badge colour/label updates immediately to match the selected layer

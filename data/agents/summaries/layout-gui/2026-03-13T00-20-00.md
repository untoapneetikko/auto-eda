# PCB import now uses library layout_example for example-circuit components

## What changed
- `backend/schematic_api.py` — `import-schematic` endpoint completely reworked for example-group components.

## How it works
When a schematic component was placed via the library "example circuit" feature it carries `_exampleGroupId` and `_exampleX/_exampleY` fields. The import-schematic endpoint now:

1. **Loads** `library/{slug}/profile.json` → `layout_example` + `example_circuit` for each unique `_exampleSlug` in the schematic
2. **Builds a position→ref map** from `example_circuit` components (keyed on their integer-rounded x/y)
3. **Pre-computes anchor PCB position** — the main IC is placed via normal schematic scaling
4. **Offsets support components** — caps, inductors etc. are positioned as `anchor_pcb + (le_pos − le_anchor_pos)`, keeping the exact layout_example topology
5. **Translates traces & vias** — pre-routed connections from the layout_example are stamped onto the board at the same offset

## Match chain
```
sc._exampleX=130, _exampleY=240
  → example_circuit c2 (ref="C2")
    → layout_example "C2" at (3.675, 8.5)
      → PCB: anchor_pos + (3.675−3.0, 8.5−5.25)
```

## Result (PMA3_83LNW example)
- U1 (anchor) → board centre
- L2, C2, C3 → 3.25mm above U1 (input side)
- L1, C1 → 3.25mm below U1 (output side)
- 2 pre-routed traces translated onto the board

- `backend/schematic_api.py` — 196 lines added

## Notes
- Gracefully falls back to schematic scaling if no layout_example exists or a component can't be matched
- Power symbols (VCC/GND) are skipped as before — no PCB component

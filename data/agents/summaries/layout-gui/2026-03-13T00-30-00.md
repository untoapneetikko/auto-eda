# Net inheritance from layout_example to PCB import

## What changed
Nets are now fully propagated from the library `layout_example` into the PCB board when importing a schematic that uses example-circuit components.

## How it works
When example-group components are detected (`_exampleGroupId` set), a net mapping chain is built:

```
LE internal net ("N1")
  → LE canonical net ("RF_IN")   via le.nets[] list
    → Schematic net ("RFIN")     via pad_net lookup after ref translation
```

This mapping is then applied to:
1. **Trace net fields** — `net: "N1"` becomes `net: "RFIN"`
2. **Via net fields** — same remapping applied
3. **Segment coordinates** — fixed a bug where traces stored as `segments: [{start, end}]` were not being offset at all (only x1/y1/x2/y2 and points[] formats were handled before)
4. **pcb_nets list** — LE canonical nets translated to schematic names with translated pad refs are merged in, so ratsnest and DRC see full pad-level connectivity

## Result for PMA3_83LNW example
| LE canonical | SC net | Pads |
|---|---|---|
| GND | GND | U1.1, U1.3…, C1.2, L1.2 |
| RF_IN | RFIN | U1.2, L1.1, C2.1 |
| RF_OUT_DC_IN | RFOUT | U1.8, L2.2, C3.2 |
| VDD | VDD | L2.1, C1.1 |

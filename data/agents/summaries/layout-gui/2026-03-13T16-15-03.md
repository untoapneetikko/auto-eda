# Route preview and clicks snap to minimum angle boundary

## What changed
- Added `_clampRoutePoint(ex, ey)` helper: given a raw cursor position, it computes the angle that would form at the last junction and — if it's below `DR.minTraceAngle` — rotates the endpoint onto the nearest boundary ray at exactly the minimum angle. Distance from the junction is preserved.
- `_drawActiveRoute`: live dashed preview now runs through `_clampRoutePoint`, so the ghost segment visually sticks to the minimum-angle boundary and never shows a forbidden bend.
- Click handler: committed route points also go through `_clampRoutePoint`, so clicking inside the forbidden zone places the point on the boundary instead of rejecting it. The old error message is gone.

## Files
- `frontend/static/js/pcb-editor.js`

## Notes
- The minimum is still `DR.minTraceAngle = 90` (set last change). Adjust that value to change behaviour.
- Pad-snap still works: if the cursor is over a same-net pad the pad position is used first, then clamped if needed.

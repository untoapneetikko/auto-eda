# Wires and vias are now part of groups

## What changed
- **`groupSelected()`** — after grouping components, automatically assigns the same `groupId` to any trace whose entire geometry falls within the group's bounding box (+2 mm margin), and to any via inside that bounding box.
- **`ungroupSelected()`** — now also clears `groupId` from traces/vias that belonged to the dissolved group.
- **`_drawGroups()`** — group bounding box now expands to include grouped trace segments and via positions. Active-group highlight also triggers when a grouped trace or via is selected.
- **Pointer-down handler** — clicking on a trace or via that has a `groupId` now selects all the group's components and starts a group drag instead of dragging just that wire/via.
- **Group drag (mousemove)** — when dragging a grouped component, ALL traces with the same `groupId` are translated by `dx/dy` each frame, and grouped vias too. Non-grouped drags continue using the existing pad-snap endpoint logic.

# Fixed duplicate V1 board creation on PCB import

## What changed
- Added `_importingProjectId` guard in `pcb.html` message handler for `importProject`.
- When a second `importProject` arrives for the same project while the first is still processing, it is now silently ignored.
- The guard is released in a `finally` block so subsequent imports (e.g. re-opening PCB later) always work correctly.

## Root cause
`_doPCBImport` in `index.html` retries every 400ms. The async `importProject` handler in `pcb.html` takes ~800–1000ms to complete (fetch existing boards → import schematic → save). So the retry fires while the first handler is still mid-flight. Both handlers see zero existing boards → both call `import-schematic` → backend assigns "V1" to both → two boards saved, both named "V1".

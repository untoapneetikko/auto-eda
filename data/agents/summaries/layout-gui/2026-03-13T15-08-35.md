# Layer management + solder paste/mask layers added to Component Layout editor

## What changed
- **`FootprintEditor` class** (`index-footprint.js`) now has a `layers` object with 6 layers (Courtyard, F.Fab, F.Paste, F.Mask, F.Cu, F.SilkS), mirroring the PCB editor exactly
- New `_padLayer(pad, i, layer)` method renders solder mask and paste shapes:
  - **F.Mask**: pad outline expanded by `maskExpansion` (default 0.10 mm) on every side — applies to all pad types
  - **F.Paste**: pad outline reduced by `pasteGap` (default 0.05 mm) on every side — SMD pads only (thru-hole pads get no paste)
- `_render()` rewritten to draw in correct layer order: Courtyard → F.Fab → F.Paste → F.Mask → F.Cu → F.SilkS
- New `buildFPLayerPanel()` function: colour dot (with inline colour picker), layer name, eye toggle — same UX as PCB editor's `buildLayerPanel()`
- **Component Layout tab** (`index-library.js`) gains a third column with:
  - Layer panel (`#fp-layer-list`)
  - **Mask Expansion** number input (mm) — live-updates the canvas
  - **Paste Gap** number input (mm) — live-updates the canvas
- SVG footer bar now shows current mask/paste values

## Files
- `frontend/static/js/index-footprint.js`
- `frontend/static/js/index-library.js`

## Notes
- Paste gap only applies to SMD pads — thru-hole pads have no paste stencil opening (correct KiCad behaviour)
- Mask expansion / paste gap values persist on the `fpEditor` instance while the tab is open
- Hard-refresh (Ctrl+Shift+R) required to see changes in the browser
